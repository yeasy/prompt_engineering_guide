# 第十二章：自动化提示词工程

## 12.1 自动化提示词生成技术

自动化提示词生成让机器参与提示词的设计和优化过程，减少人工成本。

### APE：自动提示词工程

APE（Automatic Prompt Engineering）使用模型本身来生成和优化提示词：

```
流程：
1. 定义任务目标和评估指标
2. 让模型生成多个候选提示词
3. 在测试集上评估各候选
4. 选择最佳提示词
5. 迭代优化
```

### 元提示（Meta-Prompting）

使用提示词来生成提示词：

```
你是一位提示词工程专家。

任务描述：
{task_description}

目标：
{expected_outcome}

请为这个任务设计一个有效的提示词模板，包括：
1. 角色设定
2. 任务指令
3. 格式要求
4. 示例（如需要）
```

### 基于反馈的优化

根据输出结果反馈改进提示词：

```
原始提示词：{original_prompt}
输出结果：{output}
问题反馈：{feedback}

请基于反馈优化提示词，解决发现的问题。
```

### 小结

自动化提示词生成是提示词工程规模化的重要方向。APE（自动提示词工程）框架通过让模型自动生成和评估候选提示词，显著降低了人工设计成本。元提示技术将提示词设计本身变成一个可自动化的任务，而基于反馈的优化则实现了持续改进的闭环。

在实际应用中，建议结合人工专家知识和自动化工具：先用自动化方法快速探索候选空间，再由人工进行精细调优和质量把控。后续章节将介绍更多优化工具和评估体系。
